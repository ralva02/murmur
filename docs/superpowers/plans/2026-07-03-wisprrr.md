# Wisprrr Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A local, free, single-user macOS menu-bar voice-to-text app: hold a hotkey, speak, and cleaned/formatted text is inserted at the cursor in whatever app is focused.

**Architecture:** Two SPM targets. `WisprrrCore` is a pure-logic, fully unit-tested library (models, transcript processing, prompt building, Ollama client, JSON stores, diff). `Wisprrr` is the AppKit/SwiftUI executable holding OS adapters (CGEventTap hotkeys, AVAudioEngine + SpeechTranscriber ASR, AX context reading and text injection) orchestrated by a `DictationController` state machine. A `Scripts/make_app.sh` assembles a launchable `.app` bundle.

**Tech Stack:** Swift 6 / SPM, swift-testing (`import Testing`), AppKit + SwiftUI, Speech (`SpeechAnalyzer`/`SpeechTranscriber`, macOS 26), AVFAudio, ApplicationServices (AX), Ollama HTTP API (`gemma4:e4b`).

**Conventions:** TDD for every WisprrrCore task. OS-adapter tasks (5–9) are verified by `swift build` plus the `--process-text` CLI path and manual smoke; their code blocks are complete starting implementations to be adjusted against the installed SDK if the compiler disagrees. Commit after every task.
